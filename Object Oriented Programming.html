Object Oriented Programming

A quick guide to understanding Object Oriented Programming and its usages.

What is Object Oriented Programming?
Object Oriented Programming (OOP) is a method of programming which is focused on the creation of objects which contain data and have usable methods to carry out functions in a organized manner, and one that is more efficient.

Definitions

  • Class: A description of an object type. While multiple objects can be created using this class, each object can have differentiating characters.
  • Object: An object is an instance of a class that has an identity (Defines each object which distinguishes it from other objects), state (Values stored in variables which        may be used to define the object), and a behavior (Methods which allow for specific functions to be carried out).
  • Constructor: In order to create a new object in a class, constructors are used. Constructors have the same name as the class they are used in. They often contain        parameters (values) that are stored in variables which are then stored as part of each object that is created using the constructor. Thus, several objects may have a        few identical values but can also contain other distinguishing attributes

    • Class Structure - How it is all connected:
    To create a program following an OOP structure, the class is declared first. This is followed by a constructor, with the same name as the class, which may or may not contain parameters. A constructor with no parameters is called a default constructor. When the constructor is created with the desired attributes, object defined by this constructor can then be created.

    Java Inheritance:
    Inheritance occurs in Java programming similar to as it does in real life. Classes can inherit properties/methods from other classes. The initial class is called the Parent Class/Superclass, and the class that inherits from it is called the Child Class/Subclass. While the parent class can only access its own methods, the subclass accesses all of its own methods as well as all of the methods of the superclass. Note that constructors are not inherited, however they can still be accessed using the keyword “super”.
